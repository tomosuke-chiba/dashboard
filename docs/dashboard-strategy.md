# 採用ダッシュボード戦略ドキュメント

## 目的

クライアントの**採用成功率100%を達成**するためのダッシュボード。

**採用成功の定義**：
> クリニック別の目標採用人数を、契約から12ヶ月以内に全員採用すること

**ダッシュボードのゴール**：
> 各媒体での重要な指標を把握し、どのような問題が起きているかを一目で確認でき、その問題を改善するための施策が明確にわかるダッシュボードを提供する

---

## 対象媒体と特性

| 媒体 | 採用経路 | コスト構造 | 追うべき指標 |
|------|----------|-----------|-------------|
| **GUPPY** | スカウト + 閲覧 | 閲覧100円、スカウト1000円 | 両方 |
| **ジョブメドレー** | スカウト + 閲覧 | スカウト月200通無料 | 両方 |
| **Quacareer** | スカウトのみ | - | スカウトのみ |

### 重要な考え方

#### GUPPYの戦略
- **閲覧数の最大化ではなく、応募率の最大化を優先**
- 閲覧が増えても応募につながらなければコストだけかかる
- 正しい順序：① 応募率を高める → ② その上で閲覧数を増やす → ③ 閲覧率が異常に高い（>30%）場合は不正アクセスを疑う

#### ジョブメドレーの戦略
- **検索順位が最重要**（順位が低いと露出そのものが減る）
- スカウトは月200通無料なので積極的に送る

#### Quacareerの戦略
- **スカウトのみ**（閲覧での上位表示はコントロール不可）
- 開封率と応募転換率を重視

## 採用ファネル構造

```
【GUPPY・ジョブメドレー】
露出（検索表示・掲載）
    ↓  ← 閲覧率
興味（詳細閲覧）
    ↓  ← 応募率
応募
    ↓  ← （媒体外）
採用決定

【Quacareer】
スカウト送信
    ↓  ← 開封率
スカウト開封
    ↓  ← 応募転換率
応募
    ↓  ← （媒体外）
採用決定
```

**データ取得範囲**：
- 媒体から自動取得できるデータ
- 手入力で補完するデータ（スカウト送信数など）

|---------------|-------------------|------|

| 閲覧率が低い（<5%） | ①求人タイトルを変更（具体的な給与・勤務地を明記） ②メイン写真を変更（明るく、魅力的なものに） | タイトルと写真でクリックされるかが決まる |
| 閲覧率が異常に高い（>30%） | ①GUPPY運営に報告 ②一時的に様子見（1週間） ③改善なければ掲載停止検討 | 不正アクセスの可能性、コスト増大リスク |
| 応募率が低い（<1%） | ①求人詳細の写真を追加（5枚以上） ②福利厚生欄を充実 ③職場の雰囲気がわかる文章に書き換え | 詳細を見ても魅力が伝わっていない |

### ジョブメドレー（閲覧経路）

| 発生しうる問題 | 現場の解決策（仮説） | 備考 |
|---------------|-------------------|------|
| 検索順位が低い（>20位 or 圏外） | ①**【最重要】求人の「特徴」を漏れなくチェック（該当するものは全て）** ②**定期更新（月1回以上、写真更新＋訴求文見直し）** ③**スピード返信（24時間以内）で「スピード返信」アイコンを表示** ④**写真を充実（最低4枚：外観/内観/スタッフ/設備）** ⑤**「職員の声」「職場の環境」を登録** ⑥給与情報を充実（詳細に記載） ⑦タイトルを見直し ⑧採用単価をアップ | 特徴タグ未設定は検索結果に乗らない可能性あり。更新・返信速度・写真が順位に直結 |
| 閲覧数が少ない（日<10件） | 検索順位を改善（上記施策を実行） | 順位と連動している |
| 求人ページ応募率が低い（<0.8%） | ①写真を充実（1枚目は横1200px×縦675px以上、明るく魅力的に） ②「職員の声」「職場の環境」を登録 ③勤務イメージがわかる説明文に変更 ④「1日の流れ」などを追記 | 写真の質と職場情報の充実度が重要 |
| 原稿更新からの日数が長い（>30日） | ①すぐに原稿更新（写真1枚追加・訴求文見直しでOK） ②月1回更新をカレンダーに登録 | 1ヶ月以上更新なしは「古い求人」と認識される |
| 「スピード返信」アイコンがない | ①応募後24時間以内の返信を徹底 ②返信テンプレートを用意して即対応できる体制を作る | 特集ページ掲載＋信頼性アップ |

#### ジョブメドレー検索順位改善の5大重要ポイント

1. **求人の「特徴」を漏れなくチェック**
   - 求職者は検索時に「特徴」から条件指定する
   - 該当する特徴は全部チェック（複数登録OK）
   - これが抜けていると、そもそも検索結果に乗らない・絞り込みで消える

2. **定期更新（目安：1ヶ月に1回）**
   - 求職者が求人を"最新"と認識するのは約1ヶ月
   - 求人一覧は「新着順」表示にもできるため、更新＝上位表示に寄与
   - 更新箇所がないなら 写真更新 ＋ 訴求文タイトル／本文見直し が推奨

3. **スピード返信（24時間以内）**
   - 応募後のやり取りで24時間以内返信を継続すると
   - 求人ページ・検索一覧に「スピード返信」アイコンが表示
   - スピード返信求人の特集ページにも掲載され、露出が増える

4. **写真の充実（最低4枚＋1枚目が勝負）**
   - 「外観／内観／スタッフ集合／設備」4種が揃う求人は、写真なしより応募が集まりやすい
   - 1枚目は一覧にも出る＝最重要
   - 推奨サイズ：横1200px × 縦675px以上
   - 暗い・雑多・楽しそうじゃない集合写真は逆効果

5. **「職員の声」「職場の環境」を登録**
   - どちらも管理画面から登録でき、掲載すると求職者の理解が進む
   - それらを掲載している求人の特集ページにも載り、露出が増える

## 運用方針

### フェーズ1：仮説ベースで運用開始
- 上記KPI閾値は仮説であり、運用しながら調整
- 実際のデータを1ヶ月蓄積後、閾値を見直し

### フェーズ2：データに基づく最適化
- 「この施策で改善した」事例を蓄積
- 効果のあった施策を対応表に反映
- 閾値を実データに基づいて調整

### フェーズ3：継続的な改善
- クリニックごとのベストプラクティスを共有
- 成功パターンをテンプレート化
- 新しい施策を継続的に追加

## ダッシュボード機能要件

### 必須機能
1. **KPIサマリーカード**（色分け＋アラート表示）
2. **目標採用進捗カード**（クリニック別・職種別）
3. **目標設定機能**（手動入力）
4. **アラート一覧**（改善推奨・優良指標の両方）
5. **既存データの色付け**（閾値に基づく背景色変更）

### 推奨機能（将来追加検討）
- クリニック全体統合ページ（全媒体の状況を一覧）
- 施策実行チェックリスト
- 過去データとの比較グラフ
- 改善施策の実行履歴管理
